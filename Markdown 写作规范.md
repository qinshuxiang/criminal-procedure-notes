# 个人 Markdown 写作规范

> 适用环境：Typora · 本地存档 · 无需兼容其他平台

## 一、文件头部（YAML Front Matter）

所有文章必须以如下 YAML 头开始，不可省略。字段名固定，冒号后留一个空格。

```javascript
---
created: 2026-05-16 14:30:00
modified: 2026-05-16 15:20:00
summary: 本文围绕某个核心问题，从A、B、C三个角度展开，最终得出D结论。
---
```

- `created` ：文件首次创建的日期时间，手动填入。
- `modified` ：最近一次修改的日期时间，每次存盘前手动更新。
- `summary` ：全文核心概括，写成一句完整的陈述，不用列表。

## 二、标题层级

文件本身即为一级标题（话题），**正文内 只出现一次 `#`，就是文章首行 `#文件名称`** 。所有标题从 `##` 开始，可下探至 `###`。不再使用更深的标题级别。

- 长文用 `##` 划分大板块，用 `###` 细化内部逻辑。
- 标题文字内 **禁止出现数字编号**（如 “1.”“一、” 等），标题本身应直接描述内容。

```javascript
## 问题的起源

### 第一次遇见的场景
```

若需要更轻的语义分割，使用分隔线 `---` ，**前后各留一个空行**：

```javascript
一段正文结束。

---

另一层意思开始。
```

## 三、段落与换行

**核心原则：一个段落只表达一个完整观点。**  
理想段落字数约 250 字，允许远超 5~8 行，源文件中就是一个长长的一行，绝不在句子中间按回车。

- 段落之间 **有且仅有一个空行**，不使用多个空行拉伸间距，也不在段落前加缩进。
- 格式从视觉到结构都保持 “缩紧” ：没有装饰性空行，没有多余符号，一切服务于文字的连贯与密度。

**极少数节奏换行的处理：**  
如果某一句为了语气或节奏必须在视觉上另起一行（但仍属同一段落），在该句末尾输入 **两个空格** 再回车。渲染效果为同一段落内换行。

```javascript
这是第一句。  
这一句因为语气的停顿而另起一行，但它还是这个段落的一部分。
```

日常写作尽量避免这种换行，维持长段落的基本形态。

## 四、序号与列表

**全文禁止使用有序列表 `1. 2. 3.` 和无序列表 `-` 或 `*`。**  
需要列出要点或步骤时，一律写成自然段落，在段内用 **“一、……二、……三、……”** 的形式推进，让条目融入文气。

```javascript
当时我想到了三种可能。一，他根本没有收到信息；二，收到信息但故意不回；三，回了信息却被系统过滤。这三种可能性指向的后续动作完全不同。
```

若确实需要罗列参考来源等辅助信息，使用 **脚注** 方式，绝不开列清单。（见第七条）

## 五、强调

强调文字 **只用 `**粗体**`** ，不使用斜体，也不使用粗斜体、高亮等其它形式。  
粗体应用于真正需要视觉停留的关键词、核心结论，控制密度，一段内不超过两处。

## 六、链接

所有可点击链接统一使用 **行内式** ：`[显示文本](URL)`。  
如果链接是对某个说法的出处引用，改用 **脚注** ，不在正文后罗列网址。

```javascript
正文中提到某数据[^1]，读者可以自行核对。

[^1]: 国家统计局，2025年年度报告，https://www.example.com
```

脚注标识用方括号内 `^` 加数字或简短英文，文末脚注定义独立成段，不视为列表。Typora 会自动渲染为脚注。

## 七、引用块

引用块 `>` 用于三类内容：**对话、引文、补充性的解释说明**。  
短促的对话或引文，单独设块；说明性内容也可以整段放入，使正文保持长段落的纯粹。

```javascript
> 他当时只回了一句：“别等了。”
```

引用块内同样遵守段落规范：一段一个完整观点，不使用列表。

## 八、图片与配文

所有图片必须 **居中**，并附带配文。图片文件名遵循 `文件名称_相关内容` 的格式。

```javascript
<div align="center">
<img src="文章标题_实验示意图.png" alt="实验示意图" />
*图：初期实验装置的简化结构*
</div>
```

- 用 `<div align="center">` 包裹确保在 Typora 中居中。
- 图片描述写在 `<img>` 的 `alt` 属性中。
- 配文另起一行，用斜体，放置于图片下方居中。

## 九、代码与行内代码

技术内容中，行内代码用单反引号 `` ` `` 包裹，代码块指定语言统一为 `javascript`。即使代码框内容本身是 Markdown、YAML、HTML 或其他格式，也统一写作 ` ```javascript `，保持网页中的代码高亮风格一致。

```javascript
function coreLogic(input) {
  return input.map(item => item.value * 2);
}
```

即使不是 JavaScript，只要属于代码逻辑展示，也优先以此风格标记，保持颜色高亮的一致性。行内代码前后不与中文之间空格（依个人习惯，本规范不作强制）。

## 十、写作流程与辅助

1. 新建文件时，先填入 YAML 头部，文件名即一级标题。
2. 用 `##` 写出文章骨架，再用 `###` 细化，其余过渡用 `---` 分隔线。
3. 严格保持长段落写作，段落之间只空一行，全文无列表。
4. 需要注明来源时，用脚注而非列举。
5. 每次修改后，手动更新 `modified` 时间，确保 front matter 准确。
